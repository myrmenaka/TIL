## 代入と出力  
`変数`　→　データ（値）を入れる箱  

`$変数名 ⁼ 値;`  

`$`は変数であることを表す  
変数に値を入れることを`代入`という  
イコール記号（＝）を境に、右→左の順で読む  
実行した結果を見ることを`出力`という  

単一のデータを保存する変数を`スカラー変数`という  

## 出力方法と連結方法  

`print命令（プリント）`　変数と値の連結にドット記号（.）を使う  
`echo命令（エコー）`　変数と値の連結にドット記号とカンマ（,）を使う  

ドット→繋げた結果を出力  
カンマ→変数と値をそれぞれ順々に出力  
※ドットの方が連結コストがかかるため、カンマより若干出力が遅い  

## データ型の出力  

`ver_dimp（ヴァーダンプ）`　→　データ型と値を出力  
`gettype（ゲットタイプ）`　→　データ型のみ出力  

## 変数の命名ルール  

- 文字とアンダースコアのみ（半角英数字）
- 大文字と小文字は区別される
- 数字で始まってはいけない
- 2つ以上の単語の構成は、2つ目の単語の頭文字を大文字にする

## データ型  

- `文字列型`：文字の集合  
- `整数型`：整数  
- `浮動小数点数型`：小数  
- `論理型（真偽値型）`：true（トゥルー）（真）、false（フォルス）（偽）で表す
- `NULL型（ヌル）`：未定義の値（存在しないこと）
- `オブジェクト型`：クラスのインスタンス
- `リソース型`：外部リソースへの参照

・文字列で出力する際には、シングルクォート（'）またはダブルクォート（"）で囲む  
  変数の値を出力する場合は直接記述する  
・論理型、NULL型、空文字（""）はechoでは目に見えない  
  →データ型の出力を使う  

※空文字とNULL    
  前知識：`文字`＝記号としての文字、`文字列`＝文章としての文字
  空文字は、文字列型の変数に対してのみ使う  
  
  `NULL`　＝　何もない状態  
  `空文字`　＝　何もない文字列  

  ※プログラミング言語によっては空文字がNULL扱いされる  

## 固定値と変数値の合成  

1. ドット記号（.）＝`文字列演算子`を使う  
2. ダブルクォート（"）内で、変数を`波括弧（{}）`で囲み変換する  

## 記号の出力  

意味を持つ記号類（シングルクォートやダブルクォートなど）の出力は、文法の意味を打ち消すための`エスケープ処理`をおこなう  
※PHPでは、記号の直前にバックスラッシュ（\）を付ける  

## エスケープシーケンス  

`エスケープシーケンス`　＝　エスケープ文字を先頭とする特別な意味を持つ文字列の組み合わせ  
`エスケープ文字`　＝　特別な意味を持つ文字や記号を表現するための文字  

※エスケープシーケンスはダブルクォート内で囲まれた中で使える  

`\n`　＝　ラインフィード（LF）  
`\r`　＝　キャリッジリターン（CR）  
`\t`　＝　タブ（TAB）  

※LFとCR
  制御文字→コンピュータに特定の指示を出すための文字  
  LF　＝　次の行に移る（カーソルを下に落とす）  
  CR　＝　先頭にカーソルを戻す  

※PHPの`\r`　補足
  `\r`の後の文字列は先頭の文字列の上に重ねられる  

## OSによる改行コード  

Windows　→　CR　＋　LF　（2バイト）　PHPで表すと　`\n\r`  
MacOS、Linux　→　LF　（1バイト）　PHPで表すと　`\n`  

## PHP_EOL  

PHPの定義済み定数  
OSに応じて自動的に改行文字を設定してくれる  
→　移植性を高めたいときに有効  
※シングルクォート、ダブルクォート内では使えない  
  ドット記号で値と連結する  

## ヒアドキュメント  

長い文字列に使用する  
改行や記号がそのまま入力でき、変数も展開される  

><<<開始ID　　…　　終了ID;  
>echo 開始ID;

開始IDと終了IDは一致させる  
変数に代入可能  

※終了IDは文字列本体よりも奥にインデントしてはいけない  
※タブとスペースを混ぜてインデントしてはいけない  
※文字列本体にIDと一致する文字列を含めるとエラーになる  

開始IDをシングルクォートで囲むと変数が展開されなくなる　→　`NowDoc（ナウドック）`  

※NowDocの使用場面  
・静的な文字列を定義するとき  
    変数の展開やエスケープシーケンスを適用しないとき  
・文字列リテラルの内容をそのまま扱うとき  
    特に、文字列の中にシングルクォートが含まれるときに便利  
    `文字列リテラル＝ソースコードの中に直接書きこんである文字列`  
・セキュリティを意識するとき  
    意図しないエスケープ処理や変数の展開を避けたいとき  

## キャスト  

強制的にデータ型を変更できる  

`型キャスト`  
キャストする変数の前に置いて、値を別のデータ型に変換する  

`(int)イント`：整数型  
`(bool)ブール`：論理型  
`(float)フロート`：浮動小数点数型  
`(string)ストリング`：文字列型  
`(array)アレイ`：配列型  
`(object)オブジェクト`：オブジェクト型  
`(unset)アンセット`：NULL型  

※ここに書いたもの以外は、エイリアス  
  PHP8.0.0以降は推奨されなくなった  

### データ型変換の関数  

`intval()イントバル`：整数型  
`floatval()フロートバル`：浮動小数点数型  
`strval()ストール`：文字列型  
`boolval()ブルバル`：論理型  

※少数から整数へのキャストは基本切り捨てである  
  四捨五入の場合の関数　→　`round()ラウンド`  
  切り上げの場合の関数　→　`ceil()シール`  
  切り捨ての場合の関数　→　`floor()フロア`  

※論理型への変換  
  0、NULL、空文字　→　false  
  それ以外　→　true  

型キャストは高速に動作するが、関数の方が見やすい  

※整数に変換する`intval()`と`floor()`の違い  
`intval()`  
あらゆる型の値を整数値に変換する  
数字以外の文字や空文字は数値に変換できず、0が返ってくる  
`floor()`  
数値のみを扱う
※負の数の切り捨ては数値を基準にされる（例：-3.14 → -4）
  絶対値を基準にして切り捨てたい場合（-3.14の切り捨てが-3であってほしい場合）は型キャストの(int)を使う
