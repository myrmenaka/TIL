# HTML の基本
HTMLは“意味”を表す  
見た目はCSS、動きはJavaScript  

## HTML構造の3階層モデル（ブロック要素）

ブロック要素：ページの骨組みを作る要素  
→ 縦に積まれていく  
インライン要素：横に並ぶ  
※ HTML5では”ブロック/インライン”という分類は厳密にはCSSの振る舞い  
　→ HTML5では厳密な分類はない  

### 1. 大枠（ページ全体の骨格）
・役割：ページの主要な構造を示す  
・主な要素：  
    `<header>`：ロゴ・ナビ  
    `<nav>`：主要なナビゲーション  
    `<main>`：ページの主役（1つだけ）  
    `<footer>`：コピーライトなど    
・ポイント：ここは「ページの基本的な区切り」なので 意味のある要素（セマンティックタグ） を使う

```html
<header>【ヘッダー】</header>
<nav>【ナビゲーター】</nav>
<main>【メインコンテンツ】</main>
<footer>【フッター】</footer>
```

#### ■ nav は「主要なナビゲーション」に使う
・グローバルナビ（サイト全体のメニュー）  
・サイドバーのメニュー  
・フッターの主要リンク集  
・パンくずリスト（場合によって）  

＜ nav を使う条件 ＞  
・そのリンク集が「ページ移動のための主要な案内」になっている  
・ただのリンクではなく、ナビゲーションとして意味がある  

#### ■ main は 1 ページに 1 つだけ
main が表すもの
・記事本文  
・商品一覧  
・アプリのメイン画面  
・フォーム本体  
 ページの“中心となる内容”を包むのが main。  
だから 1 ページに 1 個だけ。


### 2. 中枠（トピックや章のまとまり）
・役割：メインの中で「意味のあるまとまり」を作る  
・主な要素：  
    `<section>`：章・トピック  
    `<article>`：独立した記事  
    `<aside>`：補足情報  
・ポイント：見出し（h2/h3）が付けられる内容なら section/article を使う  
→ 「このブロックは何について？」と説明できるなら section

```html
<main>
    <section>
        <h2>商品一覧</h2>
        <p>ここに商品の説明やリストを配置</p>
    </section>

    <aside>
        <h2>関連情報</h2>
        <p>広告や補足リンクなど</p>
    </aside>
</main>
```
### 3. 小枠（レイアウトや装飾のための箱）
・役割：意味はないけど、CSSで見た目を整えるために使う  
・主な要素：  
    `<div>`：意味のない箱（CSS 用）  
・ポイント：意味がないグループ化 → div    
→ 余白調整、横並び、背景色などのために使う

```html
<section>
    <h2>商品一覧</h2>
    <div class="items">
        <article class="card">商品A</article>
        <article class="card">商品B</article>
    </div>
</section>
```
### まとめ：大枠・中枠・小枠の判断基準
・大枠 → ページ全体の骨格（header/main/footer/nav）  
・中枠 → 意味のあるまとまり（section/article/aside）  
・小枠 → 意味はないがレイアウト調整に必要（div）  

✔ 見出しを付けられるなら section、付けられないなら div  
✔ 意味のある構造はセマンティックタグ、意味のない箱は div  

## 意味の早見表（セマンティックタグ一覧）
”セマンティックタグは意味を伝えるもの”で、見た目はCSSで決める  

| 要素 | 意味 | 使う場面 | 判断基準 |  
|:--:|:--:|:--:|:--:|  
| main | ページの主役 | 記事本文、商品一覧 | ページの中心は1つだけ |  
| header | 上部のヘッダー | ロゴ、タイトル、ナビ | ページや記事の“頭” |  
| nav | ナビゲーション | メニュー、パンくず | 主要な移動リンクか |  
| section | 章・トピック | 導入、特徴、まとめ | 見出しが付けられる |  
| article | 独立した記事 | ブログ記事、商品カード | 単体で意味が通じる |  
| aside | 補足情報 | 広告、関連リンク | 主役ではない情報 |  
| footer | 下部のフッター | コピーライト、SNS |	ページや記事の“締め” |  
| div | 意味なしの箱 | レイアウト、CSS用 | 意味がないなら div |  

・main は 1 ページに 1 つだけ  
・article は独立したコンテンツ  
・section は記事の中の章  
・div は意味がないが必要な箱（レイアウト用）  
・“意味があるか？”でタグを選ぶ  

```html
<html>
└── <body>
      ├── <header>（ロゴ）
      │     └── <nav>
      │
      ├── <main>（ページ主タイトル）<h1>
      │     ├── <section>（メイントピック）<h2>
      │     │     ├── <section>（サブトピック）<h3>
      │     │     │     └── <div>（レイアウト）
      │     │     ├── <section>
      │     │     └── <section>
      │     │
      │     ├── <article>（独立した記事）<h2>
      │     │     └── <section>（サブトピック）<h3>
      │     │           └── <div>
      │     │
      │     └── <article>（独立した記事）<h2>
      │           └── <section>（サブトピック）<h3>
      │
      └── <footer>
```

※ `<main>`の中に`<section>`を入れるか`<article>`を入れるかについて  
・そのページがメイントピックとして意味を持つなら`<section>`  
・`<article>`を使うのは「独立したコンテンツ」のとき  

※ `<section>`の中に`<section>`を入れるか`<div>`を入れるかについて  
・見出しが付けられるものは `<section>`  
　→ リファレンスには`<section>`に必ず見出しが必要と明記は無い  
・見出しが付けられないものは `<div>`  

## 要素（element）とタグ（tag）の違い

・要素 ＝ タグ ＋ 内容 ＋ 終了タグ までを含む”まとまり”  
・タグ ＝ 要素を構成する”記号”の部分だけ  

### ■ class と id の違い

・class → グループ化（複数OK）  
・id    → 個体識別（1ページに1つだけ）  

class は“仲間”を作る  
id は“固有の名前”を付ける

| 項目 | class | id |  
|:--:|:--:|:--:|  
| 目的 | グループ化 | 個体識別 |  
| 使える数 | 何個でもOK | 1ページに1つだけ |  
| CSS | .class-name | #id-name |  
| JS | 複数取得（querySelectorAll） | 単体取得（getElementById） |  
| 用途 | デザイン・レイアウト | 特定の要素を一意に指定 |  
| 再利用 | できる | できない |  

## 実務での使い分け

#### ■ class を使う場面
・見た目を変えたい  
・レイアウトを整えたい  
・複数の要素に同じスタイルを当てたい  
・カード・ボタン・グリッドなどのパターン  

#### ■ id を使う場面
・ページ内リンク（アンカー）  
・JavaScript で特定の1要素だけ操作したい  
・ページに1つしかない要素  

迷ったら class を使うのが基本
