# HTML 実務チェックリスト（総まとめ）

## セマンティック（意味）チェック
### ■ HTML は“意味で書く”が大原則  
・ページの骨格に `header / main / footer / nav` を使っている  
・意味のあるまとまりに `section / article / aside` を使っている  
・見出しが付けられない箱は `div`  
・見た目のためだけにタグを選んでいない  
・`strong/em/emphasis` など意味のあるインライン要素を適切に使っている  

## 見出し階層チェック（h1〜h6）
### ■ アウトラインが崩れていないかを確認
・`h1` はページに 1 つだけ  
・`h2` は大見出し  
・`h3` は h2 の中の小見出し  
・階層飛び（h2 → h4）はしていない  
・`section` の中に見出しがある  

## アウトラインチェック
### ■ ページ全体の“目次”が正しく構造化されているか
・`section / article / nav / aside` が正しく階層を作っている  
・`article` は独立した内容になっている  
・`section` は見出しを持っている  
・`main` の中身がページの主題になっている  

## フォーム構造チェック
### ■ form は意味とアクセシビリティが命
・`form` は入れ子になっていない  
・`fieldset / legend` でグループ化している  
・`label` が必ずある（`placeholder` だけにしていない）  
・`label` と `input` が `for/id` で結びついている  
・`checkbox / radio` は内包方式で押しやすくしている  
・`name` が必ず付いている（送信される）  
・必須項目は `required` で指定している  

## input / name / id のチェック
### ■ フォームの“裏側”が正しく動くか
・`id` は `label` のため  
・`name` はサーバーに送るため  
・`radio` は `name` が同じ  
・`checkbox` は `name[]` で配列送信  
・`type` が適切（`email / tel / number / date` など）  

## アクセシビリティチェック
### ■ ユーザー全員が使えるか
・`alt` が必ずある（`img`）  
・ボタンに `aria-label` が必要な場合は付ける  
・色だけで情報を伝えていない  
・`label` が正しく機能している  
・フォーカス順（`tabindex`）が自然  

## class / id の使い分けチェック
### ■ CSS と JS が扱いやすい構造になっているか
・`class` はグループ化（複数OK）  
・`id` は固有（1ページに1つ）  
・迷ったら `class`  
・CSS は `class` を中心に書く  

## 属性チェック（グローバル / 専用属性）
### ■ 属性の意味が正しく使われているか
・グローバル属性（`class, id, title, lang`）が適切  
・専用属性（`href, src, alt, type, name`）が正しい  
・不要な属性が付いていない  
・`style` 属性を乱用していない  

## 禁止事項チェック（実務で絶対NG）
### ■ これをやるとレビューで確実に止まる
・`h1` を複数置く  
・`p` で `label` の代用  
・`div` だけでページを作る  
・`img` に `alt` がない  
・`input` に `name` がない  
・`form` の入れ子  
・見た目のために `h1/h2` を使う  
・セマンティックタグを無視して全部 `div`  

## 最終テンプレ
このチェックを通れば HTML は“正しく書けている”状態
```
【構造】header / main / footer / nav を使っている
【意味】section / article / aside を適切に使っている
【見出し】h1 は1つ、階層が正しい
【アウトライン】文章の目次が自然
【フォーム】label / id / name / fieldset / legend が正しい
【入力】type / name / id の役割が正しい
【アクセシビリティ】alt / label / aria が適切
【class/id】使い分けが正しい
【属性】グローバル属性と専用属性を理解している
【禁止事項】div乱用・h1複数・labelなし等を避ける
```
