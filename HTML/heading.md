# 見出し階層（h1〜h6 の正しい使い方）
HTML の“構造理解”の中でも特に重要  
`セマンティックHTML・SEO・アクセシビリティ・文章構造`  
すべてに関わる超コア部分  

## 見出し階層（h1〜h6）の本質

✔ h1〜h6 は「文章のアウトライン（目次）」を作るためのタグ  
✔ 見た目ではなく“意味”で選ぶ  
✔ h1 はページに1つだけ  
✔ h2 は大見出し、h3 はその中の小見出し  

→ 見出しタグは“文章の構造”を表すためのもの 
であり、  
文字を大きくするためのタグではない  

## h1〜h6 の役割（階層構造）
```
h1：ページ全体のタイトル（1つだけ）
  h2：大きな章
    h3：その中の小見出し
      h4：さらに細かい見出し
```
※ `<h5>`や`<h6>`はほぼ使用機会はない  
```
<h1>サイトのタイトル</h1>
  <h2>商品一覧</h2>
    <h3>新着商品</h3>
    <h3>人気商品</h3>
  <h2>会社情報</h2>
    <h3>会社概要</h3>
    <h3>アクセス</h3>
```
### ■ h1 はページに1つだけ（超重要）
#### ✔ h1 が表すもの
・ページの主題  
・そのページが何について書かれているか  
・SEO 的にも最重要  

#### ✔ 例
・ブログ記事 → 記事タイトル  
・商品ページ → 商品名  
・会社概要ページ → 「会社概要」  

### ■ h2 は「大きな章」
#### ✔ 例
・「特徴」  
・「料金プラン」  
・「商品一覧」  
・「会社情報」  

→ h1 の下に並ぶ“章”を表す。

### ■ h3 は「h2 の中の小見出し」
#### ✔ 例
「商品一覧（h2）」の中に  
・「新着商品（h3）」  
・「人気商品（h3）」  
という感じ。

### ■ section と見出しの関係
前にまとめた内容とつながる部分  

・section の中には基本的に見出しを置く
```html
<section>
  <h2>商品一覧</h2>
  ...
</section>
```
・見出しが付けられないなら div  

### ■ article と見出しの関係
article は「独立したコンテンツ」

#### ✔ article の中には h1 を置いてもOK
（ただしページ全体の h1 とは別）

例：ブログ記事一覧ページ
```html
<article>
  <h2>記事タイトル</h2>
</article>
```
例：ブログ記事の詳細ページ
```html
<article>
  <h1>記事タイトル</h1>
</article>
```
## よくある間違い
### ❌ h1 を複数置く
→ ページの主題が分からなくなる

### ❌ 見た目の大きさで h1/h2 を選ぶ
→ CSS で調整すべき  
→ HTML は意味で選ぶ  

### ❌ h2 の下に h4 を置く（階層飛び）
→ アウトラインが壊れる  
→ h2 → h3 → h4 の順番が基本  

## 見出し階層のテンプレ
```html
<h1>ページの主題（1つだけ）</h1>

<h2>大きな章</h2>
  <h3>章の中の小見出し</h3>
  <h3>章の中の小見出し</h3>

<h2>別の章</h2>
  <h3>小見出し</h3>
```
