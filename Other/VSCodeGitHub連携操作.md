
# VSCodeとGitHubの連携手順  

- VSCode上でGitHubの操作を行う  

# メリット  

- GUIで操作できる  
- アプリを切り替えることなくプログラミングとGit両方の操作を実行できる  

# VSCodeで扱えるGitHubの基本的な項目

- ローカルリポジトリ：ローカル（自分のPC）にあるリポジトリ  
- リモートリポジトリ：サーバー（ウェブ）上にあるリポジトリ※GitHubも含まれる  
- コミット：リポジトリへファイルの追加、変更履歴の保存をすること  
- プッシュ：ローカルリポジトリのデータをリモートリポジトリへ反映（アップロード）させること  
- プル：リモートリポジトリをローカルリポジトリへ反映（ダウンロード）させること  

# 連携方法  

※GitHubとVSCodeの初期設定は割愛する

1. ローカルリポジトリを作成  
   ローカルリポジトリとして利用するフォルダを作成  
   VSCodeで作成したフォルダを開く  
   「ファイル」→「フォルダを開く」から作成したフォルダを選ぶ  
   画面左側のGitアイコンをクリックし、「リポジトリを初期化する」ボタンをクリック
   
2. ローカルリポジトリにコミットする  
   ローカルリポジトリに指定したフォルダ内に、任意のファイルを作成する  
   VSCodeのGitアイコンに数字が表示される（この数字は、ローカルリポジトリ内で新しく変更されたファイル数）  
   画面左側に表示されたファイル名にマウスポインタを合わせ、新しく表示された「＋」ボタンをクリック  
   （ファイル名が表示されない→Gitアイコンをクリック）  
   これでステージング（コミットしたいファイルを指定すること）が完了する  
   「メッセージ欄」へ変更履歴となるコメント（任意）を入力し「コミット」アイコンをクリック  

3. リモートリポジトリを作成  
   GitHub管理画面へアクセスし、「Create Repository」ボタンをクリック  
   ※すでに他のリポジトリが作成されている場合は「New」ボタン   
   項目を入力・選択し「Create Repository」ボタンをクリック  

4. ローカルリポジトリとリモートリポジトリを紐づける  
   作成したリモートリポジトリ管理画面で、リモートリポジトリのURLをコピーする（右側にコピーボタンがある）  
   VSCodeに戻り、上部の「…」→「ターミナル」→「新しいターミナル」をクリック  
   表示されたコマンド入力欄に、以下のコマンドを入力し「Enter」を押す  

```bash
   git remote add origin 「リモートリポジトリのURL」  
```

   ※リモートリポジトリのURLは、リモートリポジトリ管理画面でコピーした内容を貼り付ける  

6. リモートリポジトリにプッシュする  
   VSCode管理画面で対象のリポジトリを開いた状態で、画面左上の「…」→「プッシュ」をクリックする  
   以下のようなメッセージが表示された場合は「OK」をクリックする  

   > ”main”ブランチにリモートブランチはありません。このブランチを公開しますか？  

   プッシュが完了したので、GitHubのリモートリポジトリ管理画面を開くと、プッシュしたファイルが確認できる

# まとめ  

- リモートリポジトリの利用のためのフォルダと、GitHubで作成したリポジトリが紐づいているため、新しくリポジトリを作成する場合はこの手順をまた行う必要がある
