# ブロックレベル要素・インライン要素・例外パターン  

HTMLタグは大きく分けて２種類に分類される  

## ブロックレベル要素  

中に他のブロックレベル要素タグやインライン要素タグの両方を  

ネストして含ませることができる  

```html
　 <section>　→ブロックレベル要素
        <section>　→ブロックレベル要素
            <h1></h1>　→インライン要素
            <p></p>　→インライン要素
            <a href=""></a>　→インライン要素
        </section>　→ブロックレベル要素
   </section>　→ブロックレベル要素

```

## インライン要素  

中に他のインライン要素タグをネストして含ませることはできるが  

ブロックレベル要素タグを含ませることはできない  

```html
　<section>　→ブロックレベル要素
        <span>　→インライン要素
            text
            <span>text</span>　→インライン要素
        </span>　→インライン要素
  </section>　→ブロックレベル要素
```

## 例外パターン  

### ダメパターン  

- ｈタグ→ブロックレベル要素  
    
    だけど、この中にブロックレベル要素は入れちゃダメ  
    
    hタグの中にインライン要素を入れるのはオッケー  
    
- ｐタグ→ブロックレベル要素  
    
    この中に他のブロックレベル要素は入れちゃダメ  
    
    インライン要素を入れるのはオッケー  
    

### オッケーパターン  

- aタグ→インライン要素  
    
    何でも入れてオッケー  
    
    aタグの中にaタグは入れない  
    

### セットで使うタグ  

- listタグ  
    
    ulタグ→順序のないリスト  
    
    liタグ→項目  
    
    どちらもブロックレベル要素だが、ulタグの次には必ずliが来なくてはならない  
    
    liタグの中には何でも入れてオッケー  
    
    ```html
       <ul>
            <li>text</li>                 ・text
            <li>text</li>                 ・text
            <li>text</li>                 ・text
    　 </ul>
    ```
    

- olタグ→順序のあるリスト  
    
    olタグの次には必ずliが来なくてはならない  
    
    あとは同上  
    
    ```html
        <ol>
            <li>text</li>                  1 text
            <li>text</li>                  2 text
            <li>text</li>                  3 text
        </ol>
    ```
    

★インデントを使うと入れ子構造が明確になるので意識する  

★必ず閉じタグを設ける  

※HTML:5では新しく「コンテンツモデル」という概念が登場  

ブロックレベル要素、インライン要素という考え方ではなく  

コンテンツモデルという枠組みでタグをカテゴライズしている仕様になった

# 属性（セレクタ）  

### 主要なもの  

```html
<a target="_blank" href="URL">text</a>
```

→リンクが別タブで開く  

```html
 <h1 id="headline">text</h1>
 <p class="text">text</p>
```

→id、class：タグに名前をつける  

CSSやJavaScript使う際に有効  

idはファイルの中に１個しか存在できない  

classはいくつでも作成できる  

JavaScriptでより高速な処理をしたい時にidが有効になるシーンが多い  

CSSをあてる場合idはほぼ使わない、なるべくclassを使う  

# part１・２まとめ  

- HTML等の言語を書くにはVSCodeのようなIDEが必要  
- HTMLはwebページの骨組みを作るマークアップ言語  
- HTMLにはたくさんの種類のタグがありテキストや画像などどんな要素を扱うかによって使い分けが必要  
    
    →webブラウザがページ全体のコンテンツを解釈するのに役立つ他、Google検索エンジン等にも影響がある  
    
- タグはネストして要素を一定のまとまりでくくることができる他、idやclass等のセレクタと呼ばれる属性を加えることでCSS（JavaScript）と組み合わせて装飾できる  
- タグにはブロックレベル要素、インライン要素、例外パターン等の一定のルールがある  
- DOCTYPE宣言、metaタグ、html、head、bodyタグ等どんなwebページにも必要な雛形となるタグがあるが、VSCodeのショートカットキー（ html:5 ）で一通り呼び出すことができる
