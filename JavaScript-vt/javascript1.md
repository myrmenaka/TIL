# JavaScript の基本構文

## プログラムの記述場所

### ■ `<body>` 内に記述

・webブラウザに表示したい内容（HTMLのタグ）などを記述  
・`<script></script>` の間に記述  
・どこでも何度でも記述可能 
※ イベント処理は`</body>` タグ直前にまとめて記述

### ■ `<head>` 内に記述

・関数の記述
・関数の実行は`<body>` 内で関数を呼び出す
・`<script></script>` の間に記述  

### ■ `<head>` 内で別ファイルを読み込む

・`.js` で別ファイルを作成し読み込む  
※ 分散開発が可能  
・他のHTML文書でも利用可能  
・`<script src="***.js"></script>` と記述

## 注意点

・大文字小文字は区別される  
・コメントの記述はJava などと同様  
・半角のみ  
・処理命令の終わりは `;`  

※ 記述ミスはエラーメッセージ表示はない

## Windowオブジェクト

webブラウザ上に表示されている全てのオブジェクトの親となるもの  
・オブジェクト階層の最上位に位置  
・実行されるタイミングで自動的に作成される  

## 表示

## 推奨：webブラウザ上に表示（HTML要素の書き換え）
```js
document.getElementById("要素のidの値").innerHTML = "メッセージ";
```
・idで指定した文字列を置き換える  
※ HTML要素のinnerHTMLプロパティを変更する  
HTMLでデータを表示する一般的な方法  

### デバック：webブラウザのコンソールに表示
```js
console.log("メッセージ");
```
・コンソール：webブラウザ上で実行されたJavaScript の結果を表示  
・webブラウザ上には何も表示されない  

■ プログラムの途中結果など、デバックの際に利用  
■ 実際にサービスを提供する場合は利用しない  

### ポップアップウィンドウ上に表示
```js
window.alert("メッセージ");
```
・注意を促したいときなど、一時的に利用  
※ ユーザーがボタンを押さないといけないので、利便性に欠ける  
・HTMLのタグは利用不可  
※ アラート画面はブラウザにより表示方法が異なる  

### テストのみ：webブラウザ上に表示（HTMLに直接書き込む）
```js
document.write("メッセージ");
```
・JavaScriptのDOMメソッドで、HTMLドキュメントの読み込み中に文字列やHTMLタグをページへ直接挿入するために使用  
主にHTMLストリームに書き込みを行い、HTMLとして解釈されるが、ページ読み込み後の使用は既存コンテンツを上書きするため、実務では推奨しない  
※ DOM（Document Object Model）：  
HTML文書を木構造（ツリー構造）としてWebブラウザ内に表現し、JavaScriptを使ってWebページの内容、構造、スタイルを動的に追加・削除・変更できるAPI  

※ ページ読み込み時にコンテンツを追加  
ただし、読み込み後に実行するとページが消えるため、テスト目的以外では避けた方が良い 
※ HTMLドキュメントがロードされた後にdocument.write()を使用すると、既存のHTMLがすべて削除される   

## 変数と式

### 命名規則

・英数字、`_` のみ  
・数字開始不可  
・大文字小文字は区別される  
・予約語の使用は不可  
・変数、関数、メソッド、プロパティはlowercamelCase（先頭小文字）での命名が慣習  
・クラス名はアッパーキャメルケース（先頭大文字）  
・定数はsnake_case  

## データ型

### ■ 基本（プリミティブ）型
| データ型 | 意味 |  
|:--:|:--:|  
| string | 文字列 |  
| number | 整数・浮動小数点数 |  
| boolean | 真偽値 |  
| null | 存在しない |  
| undefined | 変数の宣言のみ（初期化されていない状態） |  

### 参照型
| データ型 | 意味 |  
|:--:|:--:|  
| object | オブジェクトを参照する値 |  
| array | 配列を参照する値 |  
| function | 関数を参照する値 |  

## 特殊な値

### ■ NaN（Not a Number）
・数値ではないことを表す  
・isNaN()関数で確認  

### ■ null値
・「何も参照していない」状態  
・「null」はリテラルなので、変数に直接代入可能  
・isNull()関数で確認  

### ■ undefined値
・未定義値  
・明示的に値を代入していない変数の初期値
・「変数名 === undefined」あるいは「typeof === "undefined"」で判定

### ■ false判定（デフォルト）
・数値の「0」  
・NaN  
・null値  
・undefined値  
・空文字列（" "）  

## 変数の宣言
```js
var 変数名;

const 変数名;

// ECMAScript2015以降の変数の宣言
let 変数名;
```

### 変数の初期化
```js
let 変数名 = 値;
```

### ※ 実務では、constが基本

### ※ var と let の違い

・再定義のルール 
var: 同じ変数名を複数記述可能  
→ 同じ名前の変数を再定義してしまう危険あり  
let: 同じ変数名は1度のみ  
→ 再定義の際エラー  

・ブロックスコープ  
var: ブロックスコープは存在しない  
let: スコープ外で参照しようとするとエラー  

### ※ const と let の違い

・再代入できるかどうか  
const: 再代入不可  
let: 再代入できる  

・中身の変更  
→ どちらもできる  

#### 使い分け
■ 基本は const  
・変数の意図が明確になる  
・予期せぬ再代入を防げる  
・React/Vue などのフロントエンドでは特に重要  
■ 値を変える必要があるときだけ let  
・ループのカウンタ  
・状態を更新する処理  
・計算途中の一時変数
